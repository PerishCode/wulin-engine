# ADR 0038: Cooked glTF Material

- Status: Accepted
- Date: 2026-07-14
- Supersedes: None
- Superseded by: None

## Context

Experiment 0034 made externally authored Fox geometry visible through the canonical runtime,
but every object still selected one of the generated diagnostic materials. The pinned source
already contained a licensed base-color texture and a single PBR material, while imported UVs
already reached the surface resolver.

Experiment 0035 tests whether authored appearance can cross the same offline boundary without
adding a runtime image loader, another texture resource, or an archetype-specific shader path.

## Decision

- Surface material and texture layer 63 are the one accepted imported Fox material. Fixture
  materials/layers 0 through 62 retain their exact generated bytes and behavior.
- `surface-catalog` verifies the pinned Fox JSON and PNG hashes at build time. It accepts one
  material, base-color texture, image, sampler, UV set 0, default white base-color factor,
  metallic 0, roughness 0.58, repeat wrapping, linear magnification, and linear mipmapped
  minification. Any mismatch fails the build.
- Build-time PNG cooking accepts exactly 1024x1024 8-bit RGB. It performs a deterministic
  16x16 box reduction to 64x64 RGBA8 and then six deterministic 2x2 box reductions to produce
  the complete 64/32/16/8/4/2/1 mip chain.
- The strict `WLFTX001` payload owns source hashes, dimensions, material factors, mip sizes,
  and mip bytes. Runtime catalog construction decodes only this payload and verifies every
  mip hash before publishing material/layer 63.
- The existing 64-layer, 64x64, seven-mip texture array remains unchanged in size, resource
  count, descriptors, root signature, shader, upload path, and lifetime.
- The imported object cooking profile explicitly authors both archetype 7 and material 63.
  Material remains presentation authority; the renderer does not infer it from archetype or
  stable identity.
- Multiple materials or maps, alpha modes, color-space conversion, sampler variants, texture
  transforms, runtime image streaming, and source skeletal data require later experiments.

## Consequences

- The accepted imported scene now has source-authored orange, white, and dark Fox appearance
  with exact GPU/CPU UV and texel agreement.
- Material 63 has durable imported semantics and can be selected explicitly by any authored
  presentation. Content that wants the complete Fox appearance must author archetype 7 and
  material 63 together.
- PNG/glTF dependencies remain build-only. Runtime binaries, texture resources, and surface
  dispatches do not gain a decoder, source path, descriptor, or compatibility branch.
- The current byte-domain box filter intentionally does not claim general sRGB-correct texture
  processing. A broader material pipeline must supersede this single-source contract.

## Evidence

Experiment 0035 passed the direct 484.7-second workflow. The source texture cooked to exact
16,384/4,096/1,024/256/64/16/4-byte mips with catalog GPU bytes unchanged at 1,500,416.
All six visible samples selected material 63 and matched CPU texels/colors with zero channel
delta. The observed material mask contained only bit 63.

Manual capture inspection confirmed consistent orange bodies, white bellies/tail tips, and
dark regions aligned to the Fox geometry. All time, hold/failure, traversal, rollover,
64-publication resource, and 16-cycle lifecycle regressions passed without device removal.
Generated evidence is ignored under `out/captures/0035-cooked-gltf-material/`.
